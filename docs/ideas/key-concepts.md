# キーコンセプト整理

## ベイズ最適化 (BO)

「評価コストが高い関数の最適値を、なるべく少ない試行で見つける」手法。

```
普通の最適化: f(x)を何万回も評価してOK（勾配降下法等）
ベイズ最適化: f(x)の1回の評価が超高コスト（実験、シミュレーション等）
            → 少ない評価回数で賢く探索する
```

2ステップのループ:
1. **サロゲートモデル**: 入力xに対する予測値と不確実性を推定
2. **獲得関数**: 次に評価すべきxを決定（探索 vs 活用のバランス）

Optuna もBOベース（TPE）なので、コンセプトは同じ。

## MOBO（Multi-Objective Bayesian Optimization）

通常のBOは目的関数が1つ。MOBOは複数目的を同時に最適化。

パレートフロント = どの点も他の点より全目的で優れてはいない最適解の集合。

```
目的1 (TC)
  ↑
  |    *
  |   * *
  |  *   *      ← パレートフロント
  |       *
  |        *
  +----------→ 目的2 (柔軟性)
```

qNEHVI = 「この候補を評価したらパレートフロントがどれだけ広がるか」の期待値。

## BoTorch と GPyTorch の関係

```
PyTorch（深層学習基盤）
  └── GPyTorch（ガウス過程ライブラリ）
        └── BoTorch（ベイズ最適化ライブラリ）
```

| ツール | 用途 | 特徴 |
|--------|------|------|
| Optuna | ハイパラチューニング中心 | 手軽、すぐ使える |
| BoTorch | 研究〜実務のBO全般 | 柔軟、MOBO対応、カスタム可能 |

## Deep Kernel Learning (DKL)

ニューラルネットの特徴抽出力 + ガウス過程の不確実性推定を組み合わせた手法。

```
入力(分子記述子) → MLP(特徴抽出) → GP(予測+不確実性)
```

普通のGPは高次元入力が苦手だが、DKLはNNで次元圧縮してからGPに渡すので高次元に強い。

## Active Learning (AL)

モデルが「自信がない」データを優先的にラベル付け（実験/シミュレーション）するフレームワーク。

```
初期データでモデル学習 → 不確実性の高い候補を選択 → 評価 → モデル更新 → 繰り返し
```

BOの獲得関数がそのままALのサンプリング戦略になる。
